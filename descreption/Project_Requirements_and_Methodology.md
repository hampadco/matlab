## ۱. مقالات مرجع

### الف) مقاله هندی — Kumar et al. (2023)
**عنوان:** Liquefaction susceptibility using machine learning based on SPT data

- **دیتاست:** 834 رکورد روانگرایی/عدم روانگرایی بر اساس SPT
- **ورودی‌ها:** z (عمق), N1,60 (عدد نفوذ), FC (درصد ریزدانه), σv (تنش کل), σ'v (تنش موثر), amax (شتاب), Mw (بزرگا)
- **خروجی:** احتمال روانگرایی (PL) — عدد پیوسته بین ۰ و ۱
- **مدل‌ها:** DNN, CNN, RNN, LSTM, BILSTM — بهترین مدل: RNN با R²=0.906
- **روش عدم قطعیت:** FOSM (First-Order Second-Moment)
  - تولید ۱۰ دیتاست تصادفی با استفاده از COV
  - محاسبه شاخص قابلیت اطمینان: β = (μF - 1) / σF
  - محاسبه احتمال روانگرایی: PL = 1 - Φ(β)

**مقادیر COV (جدول ۱ مقاله):**

| پارامتر | بازه COV | COV انتخابی |
|---------|---------|------------|
| N (عدد نفوذ SPT) | 0.1–0.40 | **0.4** |
| FC (درصد ریزدانه) | 0.05–0.35 | **0.35** |
| σv,o (تنش کل) | 0.1–0.2 | **0.2** |
| σ'v,o (تنش موثر) | 0.05–0.2 | **0.2** |
| amax (شتاب زمین) | 0.1–0.2 | **0.2** |
| Mw (بزرگای زلزله) | 0.05–0.1 | **0.1** |

**فرمول‌های کلیدی:**
- CSR = 0.65 × (σv,o/σ'v,o) × (amax/g) × (rd/MSF) × (1/Kσ)
- CRR = exp((N1)60cs/14.1 + ((N1)60cs/126)² - ((N1)60cs/23.6)³ + ((N1)60cs/25.4)⁴ - 2.8)
- FS = CRR / CSR

### ب) مقاله ایرانی — Ghasemi & Derakhshani (2021)
**عنوان:** Uncertainty analysis of liquefaction-induced lateral spreading using fuzzy variables and genetic algorithm

- **دیتاست:** 484 رکورد گسترش جانبی (Dh) از Youd et al. (2002)
- **ورودی‌ها:** Mw, R, T15, F15, D50_15, W, S — خروجی: Dh (جابجایی جانبی — عدد پیوسته)
- **روش عدم قطعیت:** تئوری مجموعه‌های فازی + الگوریتم ژنتیک (GA)
- **نحوه اعمال عدم قطعیت:**
  1. هر ورودی → مثلث فازی (رأس = مقدار دقیق، قاعده = ±5% عدم قطعیت)
  2. Alpha-cuts: α = 0 (±5%), 0.2 (±4%), 0.4 (±3%), 0.6 (±2%), 0.8 (±1%), 1.0 (0%)
  3. در هر سطح α: GA حداکثر و حداقل خروجی مدل را پیدا می‌کند
  4. مثلث‌های فازی CC, RMSE, MAE برای مقایسه مدل‌ها رسم می‌شوند
- **نکته کلیدی:** مثلث باریک‌تر = مدل Robust-تر (کمتر تحت تاثیر عدم قطعیت)

**فرمول مدل جدید MT (14 معادله):**
```
Dh = M^p1 × R^p2 × T15^p3 × (100-F15)^p4 × (D50_15+0.1)^p5 × W^p6 × S^p7 × 10^p8
```
شرایط انتخاب معادله بر اساس درخت تصمیم (14 شاخه).

## ۵. متدولوژی تحلیل عدم قطعیت فازی (مراحل اجرا)

### مرحله ۱: فازی‌سازی ورودی‌ها (Fuzzification)
هر پارامتر ورودی به یک **مثلث فازی** تبدیل می‌شود:

```
        α=1
         /\
        /  \
       /    \
      /      \
     /________\
   Xa    Xc    Xb
   α=0         α=0
```

- **Xc** = مقدار واقعی (رأس مثلث)
- **Xa** = Xc - ΔX و **Xb** = Xc + ΔX (قاعده مثلث)
- **ΔX** = COV × Xc (بر اساس مقادیر COV مقاله هندی)

**تطبیق COV با پارامترهای پروژه:**

| پارامتر پروژه | معادل مقاله هندی | COV |
|---------------|-----------------|-----|
| Z (عمق) | z | ثابت (بدون عدم قطعیت) |
| N60 (عدد نفوذ) | N1,60 | 0.4 |
| F (درصد ریزدانه) | FC | 0.35 |
| Si (تنش کل) | σv,o | 0.2 |
| Si1 (تنش موثر) | σ'v,o | 0.2 |
| Mw (بزرگا) | Mw | 0.1 |
| a (شتاب) | amax | 0.2 |

### مرحله ۲: تعریف سطوح Alpha-Cut
مطابق مقاله ایرانی، ۶ سطح آلفا تعریف می‌شود:

| α | درصد عدم قطعیت | توضیح |
|---|---------------|-------|
| 1.0 | 0% | مقدار دقیق (رأس مثلث) |
| 0.8 | 20% از ΔX | عدم قطعیت کم |
| 0.6 | 40% از ΔX | عدم قطعیت متوسط-کم |
| 0.4 | 60% از ΔX | عدم قطعیت متوسط |
| 0.2 | 80% از ΔX | عدم قطعیت زیاد |
| 0.0 | 100% از ΔX | حداکثر عدم قطعیت (قاعده مثلث) |

در هر سطح α، بازه مجاز ورودی‌ها:
```
[Xa,α , Xb,α] = [Xc - (1-α)×ΔX , Xc + (1-α)×ΔX]
```

### مرحله ۳: یافتن کران‌های خروجی در هر سطح α
برای هر سطح α و هر مدل:
1. ورودی‌ها را در بازه‌های مجاز تغییر بده
2. مدل را روی داده‌های تغییریافته اجرا کن
3. شاخص‌های ارزیابی (OA, Precision, Recall, F1, MCC, AUC) را محاسبه کن
4. **حداقل** و **حداکثر** هر شاخص را ثبت کن

روش یافتن کران‌ها (دو گزینه):
- **روش ۱ (مقاله ایرانی):** استفاده از الگوریتم ژنتیک (GA) برای بهینه‌سازی
- **روش ۲ (ساده‌تر):** نمونه‌برداری مونت‌کارلو (تولید N نمونه تصادفی از بازه α و ثبت min/max)

### مرحله ۴: رسم مثلث‌های فازی شاخص‌های ارزیابی
برای هر شاخص (مثلاً OA)، مثلث فازی با استفاده از کران‌های به دست آمده رسم می‌شود:
- محور X: مقدار شاخص ارزیابی
- محور Y: سطح α (از ۰ تا ۱)
- سه مدل M5P، ANN و RF روی یک نمودار مقایسه می‌شوند

### مرحله ۵: تفسیر و مقایسه
- **عرض قاعده مثلث (Support):** هرچه کمتر → مدل Robust-تر
- **موقعیت رأس:** هرچه به ۱ نزدیک‌تر → مدل دقیق‌تر
- بهترین مدل: هم رأس نزدیک ۱ و هم قاعده باریک
